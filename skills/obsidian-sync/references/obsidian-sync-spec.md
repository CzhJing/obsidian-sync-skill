# Obsidian 同步规范（Skill）

本文件为 @obsidian-sync-project 的配套规范，包含：文件命名规则、目录结构、MD 文件模板、标签推断规则、日志格式模板。主流程执行时引用本文件，不重复描述。

---

## 一、命名与路径规范

| 规范项 | 详情 |
|--------|------|
| 项目根目录 | `VAULT_PATH/10-19 Projects/` |
| 项目子目录 | `NN {项目名}/`（所有层级文件平铺于此，不建子目录） |
| 日志子目录 | `NN {项目名}/01 DailyLog/` |
| 删除归档目录 | `NN {项目名}/02 Delete/` |
| 顶层文件命名 | `NN.00 {项目名}.md` |
| 日志文件命名 | `NN.01 DailyLog.md` |
| 模块文件命名 | `NN.MM {模块名}.md` |
| 服务/外部API命名 | `NN.MM.SS {服务名}.md` |
| 类/接口命名 | `NN.MM.SS.CC {类名}.md` |
| 方法命名 | `NN.MM.SS.CC.FF {方法名}.md` |
| 表命名 | `NN.MM.SS.CC.FF.TT {表名}.md` |
| JD 编号分配 | 扫描同层级现有最大编号后递增 1 |

**wikiLink 规范**（统一使用含 JD 编号的完整文件名，不含 `.md`）：
**NN、MM、SS、CC、FF、TT这些字母都替换成实际分配的JD编号**
```
✅ 顶层项目：[[NN.00 {项目名}]]
✅ 子层级：  [[NN.MM {模块名}]]、[[NN.MM.SS {服务名}]] 以此类推
❌ 错误写法：[[{项目名}]]（缺少 JD 编号）
```

## 四、日志格式模板

文件路径：`VAULT_PATH/10-19 Projects/NN {项目名}/01 DailyLog/NN.01 DailyLog.md`

```markdown
## Dev Log — {Month DD, YYYY}

### {功能模块或变更类别}
- {本次改动的具体描述，重点说明"为什么这么做"而非只记录"改了什么"}

### {另一个类别}
- {工作内容}
```

**插入规则：**
- 最新日志始终插在所有已有 `## Dev Log` 条目之前
- 当天已有日志则合并到同一日期块，不新建重复日期
- 严禁删除或修改任何历史日志条目

## 六、YAML 字段规范

| 字段 | 必填 | 说明 |
|------|------|------|
| `jd-id` | ✅ | JD 编号，如 `"NN.00"`、`"NN.MM.SS"` |
| `name` | ✅ | 层级名称 |
| `desc` | ✅ | 2–3 句描述 |
| `updateTime` | ✅ | 格式 `YYYY-MM-DD`，文件创建或更新时写入 |
| `status` | ✅ | `active` / `deleted` |
| `type` | ✅ | `A-高层系统` 或 `B-文件级细节`，子层级继承顶层 |
| `tags` | ✅ | 从代码特征推断，2–5 个 |
| `author` | ⬜ | 从注释读取，无法获取时省略整个字段 |
| `createTime` | ⬜ | 从注释读取，无法获取时省略整个字段 |

## 七、删除归档规范

源码文件删除时，对应 MD 文件**不直接删除**，执行以下操作：

1. YAML `status` 字段改为 `deleted`
2. 将文件移入 `VAULT_PATH/10-19 Projects/NN {项目名}/02 Delete/`
3. 文件顶部（YAML 下方）插入归档注释：
   ```
   > ⚠️ 此文件对应的源代码已于 {YYYY-MM-DD} 删除，当前状态：已归档
   ```
4. 实际永久删除需用户手动确认后执行
